Module omni

Omnidirectional platform.

Configuration is loaded using nvs.read("omni", parameter) calls, where the available parameters are:

  • "maxpower" Limits the maximum power output of the motors as percentage. Defaults to 80.0

  • "kf" Feed-forward parameter of the motor control in (power% / (tics/s) ). Defaults to 90/1080

  • "kp" P parameter of the PID control. Defaults to 0.01

  • "ki" I parameter of the PID control. Defaults to 0.05

  • "kd" D parameter of the PID control. Defaults to 0.0

Functions

drive (x_dot, y_dot, w_dot[, phi=0])
Move the robot.

Parameters:

  • x_dot number velocity on the x axis, in m/s
  • y_dot number velocity on the y axis, in m/s
  • w_dot number rotation angular velocity, in rad/s
  • phi number rotation of the xy axis, in rad. Defaults to 0 (default 0)
enable ([on=true])
Enable/disable motors.

Parameters:

  • on boolean if true value or omitted, power up. If false value then power down. (default true)
omni.encoder.enable (on)
Enables the encoder callback. When enabled, wheel movement will trigger omni.encoder.cb.

Parameters:

  • on boolean true value to enable, false value to disable.

Fields

max_speed
Estimated maximum speed in m/s. This is computed from the maxpower setting.
device
The native C firmware module. This can be used to access low level functionality from omni_hbridge. FIXME: docs
omni.encoder.cb
The callback module for the encoders. This is a callback list attached to the motor encoders, see cb_list. This call triggers when wheels rotate.
  • id integer the wheel identifier, in the 1..3 range
  • dir integer either 1 or -1, indicating the direction of the rotation
  • count integer a rotation counter in encoder ticks.

Usage:

    local local omni = require 'omni'
omni.encoder.cb.append( function (id, dir, count) print(id, count) end )
